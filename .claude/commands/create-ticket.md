# create-ticket

GitHub Issueを対話的に作成し、たびしぇあPJのプロジェクトボード（プロジェクト番号: 2）に自動追加します。

## 実行手順

### 0. 事前確認

以下を並行して実行してください：

- `gh auth status` で `write:project` スコープが付与されているか確認する
- `gh label list --repo tabi-bit/tabi-share` で利用可能なラベル一覧を取得する

`write:project` がない場合は、以下をユーザーに案内してから処理を中断してください：

```
! gh auth refresh -h github.com -s write:project
```

### 1. 情報収集

`$ARGUMENTS` が提供されている場合はタイトルの叩き台として活用し、推定できる項目は先に埋めてください。

#### ステップ 1-A: 領域・種類の選択

**`AskUserQuestion` ツールを使って**、以下の2問を一度に提示してください（矢印キーで選択 or "Other" で自由入力できます）:

- 質問1: 「どの領域のチケットですか？」（header: "領域"）
  - フロントエンド / バックエンド / セキュリティ / 汎用（インフラ・その他）
- 質問2: 「チケットの種類は何ですか？」（header: "種類"）
  - バグ（Something isn't working） / 新機能・改善（New feature or request） / タスク（ラベルなし）

#### ステップ 1-B: タイトル・本文の確認

まず `.github/ISSUE_TEMPLATE/task-template.md` を Read して本文のセクション構成を確認してください。
`$ARGUMENTS` や選択した領域・種類からそのテンプレートに沿った草案を作成し、ユーザーに提示して確認・修正を依頼してください。

| 項目 | 内容 |
|------|------|
| **タイトル** | 機能や問題の概要（1行） |
| **本文** | テンプレートの各セクションを埋める（バグは再現手順を前提情報に記載） |

### 2. タイトル・ラベルの決定

**タイトルのプレフィックス規則**（既存Issueの命名規則に準拠）:
- フロントエンド → `【フロントエンド】{タイトル}`
- バックエンド → `【バックエンド】{タイトル}`
- セキュリティ → `【セキュリティ】{タイトル}`
- 汎用・インフラ → プレフィックスなし

**ラベル選択**:
ステップ0で取得したラベル一覧を元に、種類に最も近いラベルを1つ選択してください。
明確に対応するラベルがない場合はラベルなしとします。

### 3. Issue作成

`.github/ISSUE_TEMPLATE/task-template.md` の frontmatter（`---` ブロック）を除いた本文をベースに、ユーザーの回答を埋め込んだ本文を作成します。
ヒアドキュメントを使って改行を保持して作成してください：

```bash
gh issue create \
  --repo tabi-bit/tabi-share \
  --title "{タイトル}" \
  --body "$(cat <<'EOF'
{テンプレートのセクションにユーザーの回答を埋め込んだ本文}
EOF
)" \
  --label "{ラベル}"   # ラベルなしの場合は --label オプションを省略
```

作成後、Issue URLとIssue番号を控えてください。

### 4. プロジェクトへの追加

Issue URLを使ってプロジェクトに追加します：

```bash
gh project item-add 2 --owner tabi-bit --url {issue_url}
```

### 5. 完了報告

以下をユーザーに報告してください：

- 作成したIssueのURL
- Issue番号
- プロジェクトへの追加結果

## 参考：既存Issueの傾向

- ゴールはチェックリスト形式（`- [ ] ...`）にすると進捗管理しやすい
- バグは再現条件・期待動作を前提情報に記載する
- 関連IssueやPRは「その他」セクションに `#番号` で記載する
