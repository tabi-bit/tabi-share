# Issue #175 · 閲覧画面ページ選択UI改善 · 設計メモ

## 背景 / 問題

- **主訴1**: 閲覧画面ではセレクトボックスがヘッダーにあり、閲覧に集中したいのに存在感がある。ヘッダー縮小時は「ヘッダー展開 → 選択UI展開 → 選択」の3ステップが必要。
- **主訴2**: モバイルではセレクトが画面上部にあり、片手操作で届きづらい。
- **主訴3**: Trip編集はタイトル横のペンアイコンから起動できるのに対し、ページ編集はヘッダー右の別アイコンで、体験が非対称。ページ名などがどこから編集すればいいかが不明瞭。

## 前提決定

- **隣接ページ切替の主導線**
  - モバイル: **スワイプ** (既存 `PageSwipeContainer`)
  - デスクトップ: **UI + キーボードショートカット** (←→)
- **新UIはインタラクティブに見える**ようchevron/hover等の視覚アフォーダンスを明示

## モバイル決定事項

### 配置 (Case B)

- 画面下端に「**ドット・現在ページ名・chevron**」を1行に統合した**pill形状のフローティングバー**
- スワイプ主導線を補助する副導線として機能
- 本文コンテンツはバーの下を通過してスクロール（absolute配置）→ バー領域はcream→透明のグラデーションフェード

### 展開UI

- 下部pillタップで**フローティングカード式のシート**が下から浮上（**フルワイズドロワーではない**）
- 左右余白 + 全周角丸 + 影で下部バーと同じフローティングリズム
- シート表示時は下部バー自体は非表示化（同じ位置にシートが現れる自然な視覚接続）

### 視覚レイヤーの分離

- **ブロック**: 白カード + border + 微shadow（現状維持）
- **下部バー / シート**: border撤去 + 強めshadowで浮遊、`border-radius: 999px` のpill形状
- 上記により、ブロックとバーが重なっても視覚的に混同しにくい

### 見た目 (フロスト)

- モバイルpillもデスクトップと揃えてフロスト仕上げ
- デスクトップより透過度は控えめ（60%程度）・blurも弱めにチューニング
  - 背後にブロック(白カード)が近づきやすいため、可読性を優先
- Android等 backdrop-filter 未対応環境では不透明白にフォールバック

### シート内容 (案IV: モード別)

- **閲覧モード**: ページ一覧のみ
- **編集モード**: ページ一覧 + 各行のページ名右に**ペンアイコン**（Trip編集と同じインラインパターン） + 末尾に「**＋ ページを追加**」

### 撤去対象

- ヘッダー右の「ページ編集」ボタン (`PageInfoEditButton`) → 削除
  - 編集導線はシート内のペンに一元化
  - ヘッダーがスリム化 → 閲覧集中の主訴に貢献

### エッジケース

- **1ページのみ**: pill全体を非表示。編集モードでは「＋ ページを追加」の入口だけ残す（形は要検討）
- **6ページ以上**: ドット表示から「2/8」の番号表記に自動切替
- **長タイトル**: pillは日付との兼ね合いで省略（現状の `M/D(曜)` プレフィックス + タイトル ellipsis を踏襲）／ Popover/シート内では全文表示。日付表示形式は実装時に調整
- **date未設定のページ**: 現状の `sortPages` (date昇順→id昇順, null末尾) を踏襲

## デスクトップ決定事項

### 位置 (案Z + 案ζ)

- UI主導線 + **キーボードショートカット** (←→) で隣接ページ切替
- **上部フロートpill**（ヘッダー直下に浮遊）で配置。モバイル下部pillの上下鏡像で視覚的統一
- pill形式: `‹ ページ名 ⌄ ›`（chevron両側 + 中央クリックでPopover展開）
- ヘッダーは Trip名+ペン + アクションのみでスリムに保つ

### 見た目 (フロスト)

- pill背景: **backdrop-filter による半透明フロスト**
  - Light: `rgba(255,255,255,0.45)` + `backdrop-filter: blur(20px) saturate(1.5)`
  - Dark: `rgba(32,31,35,0.45)` + 同blur
- 内側 1px 白 hairline（`inset 0 0 0 1px rgba(255,255,255,0.6)`）で「vibrancy」的な輪郭
- スクロールでブロックがpill裏を通過するときにフロストが効いてくる（動きで真価が出る）

### 展開UI

- 中央クリックで **Popover**（pillの下方向に開く）
- 内容はモバイルのシート案IVと同じ挙動:
  - 閲覧モード: 一覧のみ
  - 編集モード: 一覧 + 各行ページ名右のペン + 末尾に「+ ページを追加」
- ESCで閉じる、外側クリックで閉じる

### 未確定

- ヘッダー縮小挙動（`isScrolled` によるSelect→Badge化）を撤廃するか、pillを縮ませるか
- pillのスクロール追従挙動（常に浮遊?/一定量スクロールで隠す?）
- backdrop-filter 導入の全アプリへの波及（Header, Modal等へ拡げるか）
